## Introduction

**S3 Buckets** gives you S3 compatible object storage directly in the console. It uses the same standard S3 API as AWS, MinIO, and other providers, so any existing S3 client (AWS CLI, rclone, Boto3, Cyberduck, and more) works with Hippius without modification.

From this page you can create and manage buckets, upload and download objects, and generate credentials for your S3 clients.

Reach S3 Buckets from the sidebar at <BgStyledIconWithText text="Storage" icon="SidebarStorage" /> → <BgStyledIconWithText text="S3 Buckets" icon="Database" />.

For a full reference of supported S3 API operations, see [S3 Compatibility](/storage/s3/compatibility). For setup guides for specific clients, see [S3 Quickstart](/use/quickstart).

![S3 Buckets page](/img/console/s3/overview.png)

## Master Tokens

Before you can create buckets or connect any S3 client, you need a **master token**. This is your top level S3 credential, the equivalent of AWS root keys, and it has access to every bucket and every operation on your account.

The first time you open the S3 page, you'll be prompted to create one.

### Creating a Master Token

<Ordered>
  <li>Click <BgStyledText>+ Create Master Token</BgStyledText> in the page header (or click the prompt in the empty buckets table).</li>
  <li>Enter a <strong>Token Name</strong> to help you identify it later (e.g. <em>"Production Token"</em>).</li>
  <li>Pick an <strong>expiry</strong>: <strong>7 days</strong>, <strong>30 days</strong>, <strong>1 year</strong>, or a <strong>custom date</strong>.</li>
  <li>Click <BgStyledText>Create Master Token</BgStyledText>.</li>
  <li>The dialog shows your new <strong>access key</strong> and <strong>secret key</strong>. Copy both now using the copy buttons.</li>
  <li>Click <BgStyledText>Done</BgStyledText> to close.</li>
</Ordered>

![Create master token dialog](/img/console/s3/create-master-token.png)

:::warning Save your secret key immediately
Once you close this dialog, the **secret key is gone forever**. If you lose it, you'll need to revoke the token and create a new one. The access key remains visible, but the secret is never shown again.
:::

### Managing Master Tokens

Click <BgStyledText>Manage</BgStyledText> in the bucket browser toolbar to open the master tokens page. From there you can:

<Unordered>
  <li>See all your master tokens with their access keys, creation dates, and expiry dates.</li>
  <li><strong>Revoke</strong> a token immediately. Any client using it will start getting auth errors right away.</li>
  <li>Create additional master tokens (useful for rotating credentials without downtime).</li>
  <li>Drill into the sub tokens for each master.</li>
</Unordered>

![Master tokens management](/img/console/s3/master-tokens.png)

## Sub Tokens

A **sub token** is a scoped credential derived from a master token. Use sub tokens to share access without handing out your master credentials.

| Why use sub tokens | Example |
|---|---|
| **Per bucket access** | A backup script that only needs one specific bucket. |
| **Read only** | A CDN that should never be able to delete or overwrite. |
| **Time limited** | A vendor access that expires after one week. |
| **Easy revocation** | Revoke a sub token without touching the master. |

### Creating a Sub Token

<Ordered>
  <li>From the master tokens page, find the master you want to derive from and click <BgStyledText>Manage Sub Tokens</BgStyledText> on its row.</li>
  <li>Click <BgStyledText>+ Create Sub Token</BgStyledText>.</li>
  <li>Enter a <strong>Name</strong>: a label so you remember what this token is for (e.g. <em>"Production backups"</em>).</li>
  <li>Pick the <strong>Buckets</strong> this sub token can access: one, several, or all.</li>
  <li>Choose <strong>Permissions</strong>: read only (<em>Get / List</em>) or read/write (<em>Get / List / Put / Delete</em>).</li>
  <li>Set an <strong>Expiry</strong>: pick from the preset options or choose a custom date.</li>
  <li>Click <BgStyledText>Create Sub Token</BgStyledText>.</li>
  <li>Copy the access key and secret key before closing the dialog.</li>
</Ordered>

![Create sub-token dialog](/img/console/s3/create-sub-token.png)

### Managing Sub Tokens

Each sub token row shows its name, access key, bucket coverage, permission scope, and expiry date. The action menu on each row lets you:

<Unordered>
  <li><BgStyledText>View Details</BgStyledText>: open the details dialog again. The secret key is not shown again.</li>
  <li><BgStyledText>Revoke</BgStyledText>: invalidate the sub token immediately.</li>
</Unordered>

![Sub-tokens management](/img/console/s3/sub-tokens.png)

## Buckets

### Creating a Bucket

<Ordered>
  <li>With an active master token, click <BgStyledText>+ Create Bucket</BgStyledText> in the page header.</li>
  <li>Enter a <strong>bucket name</strong>. Names must be lowercase letters, numbers, and dashes only, between 3 and 63 characters, and must start and end with a letter or number.</li>
  <li>Click <BgStyledText>Create Bucket</BgStyledText>.</li>
</Ordered>

The new bucket appears at the top of the list. Click its row to open it.

![Create bucket dialog](/img/console/s3/create-bucket.png)

### Bucket Actions

Each bucket row has an action menu (three dots):

<Unordered>
  <li><BgStyledText>Open</BgStyledText>: open the bucket. Same as clicking the row.</li>
  <li><BgStyledText>Copy bucket name</BgStyledText>: copy to clipboard.</li>
  <li><BgStyledText>Delete</BgStyledText>: permanently delete the bucket. The bucket must be empty first. The dialog will block you if it still contains objects.</li>
</Unordered>

![Bucket actions menu](/img/console/s3/bucket-actions.png)

## Inside a Bucket

Clicking a bucket opens the bucket browser, where you can manage all its objects.

![Bucket detail page](/img/console/s3/bucket-detail.png)

### Folders

S3 buckets are flat by design. There are no real folders, only object keys that contain slashes. The console presents these as folders for a familiar browsing experience.

To create a folder:

<Ordered>
  <li>Click <BgStyledText>+ New Folder</BgStyledText> in the bucket toolbar.</li>
  <li>Enter the folder name. It's created relative to your current location in the breadcrumb.</li>
  <li>Click <BgStyledText>Create</BgStyledText>.</li>
</Ordered>

A zero byte placeholder object appears immediately. It's automatically removed when you upload a real object into the folder.

![Create folder dialog](/img/console/s3/create-folder.png)

### Uploading Objects

Click <BgStyledText>+ Upload</BgStyledText> to open the upload dialog. The destination is always the folder shown in the current breadcrumb. Navigate to the right folder first, then upload.

Each file must be under **100 MB**. For larger files, use an S3 client with multipart upload support. See [Connecting an S3 Client](#connecting-an-s3-client) below.

![Upload to bucket dialog](/img/console/s3/upload-object.png)

### Browsing and Previewing Objects

Click any object to preview it. Images, PDFs, and videos open inline in the browser. For all other file types, use the action menu to download the file instead.

Use the search box to filter objects by name in the current folder. Click the refresh icon to reload the listing after another client has added objects in the background.

### Object Actions

Each object has an action menu (three dots):

<Unordered>
  <li><BgStyledText>Preview</BgStyledText>: open the in browser preview.</li>
  <li><BgStyledText>Download</BgStyledText>: save the object to your device.</li>
  <li><BgStyledText>Copy URL</BgStyledText>: copy the object's S3 URL to your clipboard.</li>
  <li><BgStyledText>Details</BgStyledText>: open the details panel with full metadata (size, MIME type, last modified, key, CID).</li>
  <li><BgStyledText>Delete</BgStyledText>: permanently delete the object.</li>
</Unordered>

### Selecting and Deleting Multiple Objects

Tick row checkboxes (or the header checkbox to select all on the page) to enter selection mode. The **Selection Action Bar** at the bottom lets you bulk delete or bulk download the selected objects.

![S3 selection action bar](/img/console/s3/selection-bar.png)

## Connecting an S3 Client

Once you have a master or sub token, configure any S3 compatible client with these settings:

| Setting | Value |
|---|---|
| **Endpoint** | The Hippius S3 endpoint (see <a href="/storage/s3/compatibility">S3 Compatibility</a>). |
| **Region** | <code>auto</code>. Hippius is region less, so any value works. |
| **Access key** | From your master or sub token. |
| **Secret key** | The secret you copied at creation time. |
| **Path style addressing** | Required. Hippius does not support virtual hosted style URLs. |

Setup guides for specific clients:

## Limits

| Limit | Value |
|---|---|
| **Max single object upload via console** | **100 MB** |
| **Max bucket name length** | 63 characters |
| **Bucket name characters** | Lowercase letters, numbers, dashes |
